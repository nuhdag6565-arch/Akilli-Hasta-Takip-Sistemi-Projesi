# Akıllı Hasta Takip Sistemi - Agent Dosyası

Bu dosya, **Akıllı Hasta Takip Sistemi (Smart Patient Tracking System)** projesinin mimarisini, değişmez kurallarını ve teknik standartlarını tanımlar. Tüm yapay zeka asistanlarının bu projeyle çalışırken uyması gereken ilkelerdir.

---

## 1. Proje Tanımı

### Genel Amaç
Makine öğrenimi algoritmaları kullanarak **inme (stroke) riskini tahmin eden** ve hastalara erken uyarı sağlayan bir sağlık yönetim sistemi.

### Temel Sorun
- Dünya genelinde inme, ölüm nedenleri arasında ilk sıralarda yer almaktadır
- Mevcut sağlık sistemleri risk analizi yapmanın çoğu zaman çok geç olduğu semptomlar sonrasında yapar
- Yaş, BMI ve glikoz seviyesi gibi değişkenlerin kombinasyonu insan gözüyle analiz edilemeyecek kadar karmaşıktır

### Hedefler
- **Tahminleme Gücü:** ≥%80 doğruluk oranına sahip makine öğrenimi modeli
- **Korelasyon Analizi:** Risk faktörlerinin etkisini istatistiksel olarak belirleme
- **Erişilebilirlik:** Doktorlar ve araştırmacılar için açık kaynak hale getirme

---

## 2. Proje Kapsamı ve Veri Kaynağı

### Veri Seti
- **Kaynak:** Kaggle - "İnme (Stroke) Riski" veri seti
- **Dosya Adı:** `data/processed/temizlenmis_hasta_verisi.csv`
- **Kayıt Sayısı:** 5,110 hasta kaydı
- **Değişken Sayısı:** 12 sağlık değişkeni

### Veri Özellikleri
```
Giriş Özellikleri (Input Features):
- hasta_id          : Benzersiz hasta kimliği
- cinsiyet          : Kategorik (Erkek/Kadın)
- yas               : Sayısal (yaş)
- hipertansiyon     : İkili (0/1)
- kalp_hastaligi    : İkili (0/1)
- evli_mi           : Kategorik (Evet/Hayır/Eski/Hiç)
- calisma_tipi      : Kategorik (Çalışan/Serbest/Hükümet/İşsiz)
- ikamet_tipi       : Kategorik (Kırsal/Kentsel)
- ortalama_seker    : Sayısal (fasting blood glucose)
- vucut_kitle_indeksi : Sayısal (BMI)
- sigara_durumu     : Kategorik (Hiç/Eski/Halen)

Hedef Değişken (Target):
- inme_durumu       : İkili (0 = Risk Yok, 1 = Risk Var) [Binary Classification]
```

### Veri İşleme Kaidesi
- **Eksik Veriler:** Tespiti yapılmış, temizlenmiştir
- **Aykırı Değerler:** Filtrelenmiştir
- **Normalizasyon:** Uygulanmıştır
- **Kategorik Veri:** One-Hot Encoding ile sayısal hale dönüştürülmüştür
- **Dengeleme:** SMOTE algoritması ile veri seti dengelenmiştir (sınıf dengesizliği problemi çözmek için)

## 3. Makine Öğrenimi Mimarisi

### Seçilen Algoritma: Random Forest (Rastgele Orman)

**Seçilme Nedeni:**
- Kategorik ve sayısal verileri eşzamanlı olarak işleyebilir
- Tıbbi veri setlerinde yüksek doğruluk oranı sağlar
- Aşırı öğrenmeye (overfitting) karşı dirençli
- Özellik önem sıralaması (feature importance) sonuçları verir

**Konfigürasyon:**
- Ağaç Sayısı: 100-200 arası
- Min_samples_split: 5
- Min_samples_leaf: 2
- Max_depth: 15-20 arası

### Model Hedefleri
- **Eğitim Doğruluğu (Training Accuracy):** ≥%85
- **Test Doğruluğu (Test Accuracy):** ≥%80
- **Precision (Kesinlik):** ≥%75 (Yanlış pozitif tahminden kaçınmak önemli)
- **Recall (Duyarlılık):** ≥%75 (Riskli vakaları kaçırmamak kritiktir)

### Model Çıktısı
```python
Risk Skoru: 0.0 - 1.0 (float)
Risk Seviyesi: 
  - "Düşük" (0.0 - 0.33)
  - "Orta" (0.33 - 0.66)
  - "Yüksek" (0.66 - 1.0)
```

## 6. API Tasarımı

### REST API Uç Noktaları

#### Hasta Endpoints
```
GET    /api/hastalar              - Tüm hastaları listele
POST   /api/hastalar              - Yeni hasta ekle
GET    /api/hastalar/<hasta_id>   - Belirli hastayı getir
```

#### Doktor Endpoints
```
GET    /api/doktorlar             - Tüm doktorları listele
POST   /api/doktorlar             - Yeni doktor ekle
```

#### Risk Tahmin Endpoint
```
POST   /api/risk-tahmini
Content-Type: application/json

Request Body:
{
  "yas": 55,
  "cinsiyet": "Erkek",
  "hipertansiyon": 1,
  "kalp_hastaligi": 0,
  "evli_mi": "Evet",
  "calisma_tipi": "Çalışan",
  "ikamet_tipi": "Kentsel",
  "ortalama_seker": 140,
  "vucut_kitle_indeksi": 28.5,
  "sigara_durumu": "Halen"
}

Response:
{
  "risk_skoru": 0.75,
  "risk_seviyesi": "Yüksek",
  "oneri": "Doktora başvurun"
}
```
